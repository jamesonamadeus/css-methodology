Floating Input Labels

Proposed pattern for positioning labels on top of input areas, until user focus. At that point, label is floated above input.

If text is entered, then the label stays anchored above the input. If the input is blurred, and the user input is null, then the label will return to it's 'floated' position on top of the form input.

Question: Should this be considered an anti-pattern due to the changing position of the form element's label?
