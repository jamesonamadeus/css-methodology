Text Inputs with Descriptions

Examples of text inputs that have coupled descriptions. Namely inputs with error/warning messages and required or optional inputs that give context as to why the field is required, or an indicator that the field is optional.

Error messages should appear next to the form field that has the error to give appropriate context, and be linked to the input via an aria-describedby attribute. See the form patterns section for an alternate error pattern if the optimal pattern shown below is not possible to implement.
