Links

Utilize anchor elements to link to new documents, or to sections within a current document.

By default anchor elements that have href attributes and can receive tab focus via keyboard controls. Only in FireFox on Mac OSX does an accessibility flag need to be enabled within the system settings for a user to tab to anchor elements.

Read More Link Pattern

By their selves; "Read More", "Learn More", "View" links that typically appear at the end of short lead-ins to larger articles, do not provide enough unique context for assistive technologies.

This pattern uses the title attribute and visually hidden text to extend the visual link text to provide the needed context these links usually lack.

Mailto & Telephone Links

If a phone number or email is utilized as the visible text for a mailto or telephone link, then there is no special pattern needed.

If the phone number or email is not the visible text for either of these types of links, then be sure to add them as the value of the title attribute.

In some cases it may make sense to use the 'read more' pattern with these links, especially in situations where there are multiple points of contact and the link text is not descriptive enough for assistive technologies to differentiate between them.

Skip Links

Skip links allow users to instantly move focus from one area of a document to another. The href of these links points to a URI prefaced with a # and followed by the ID of the element to which focus should be shifted.

Hamburger Menus

Hamburger Menus can be either links or buttons depending on your no-js solution.

If used as a link, a no-js solution would require usage of the :target pseudo selector to open the menu, and a revealed close link to be a trigger to hide the menu again.

If used as a button, the hamburger menu should not exist without js, and the target menu/nav should be revealed by default, regardless of screen size.

Close

In instances where an overlay or some other type of component can obscure content, even when JavaScript is turned off, then a close trigger should be a link as opposed to a button. In these instances, on activation of the close trigger, we need to ensure that the URI points to the most logical ID to return a user's focus to.

In instances where a component can not be opened without JS, then this may as well be a button element, too.
