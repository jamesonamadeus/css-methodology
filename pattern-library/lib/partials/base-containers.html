Containers

Standard container classes we can use to ensure consistent content layout and grouping.

.page

The .page allows for top level padding, setting of a maximum width and ensuring that any off-screen content elements won't trigger horizontal browser scrolling.

.content-area(s)

content-area

Content areas allow for uniform padding and max-width

Last direct child of a content-area has a margin-bottom: 0

content-area + content-area

When content areas directly follow a previous content area in the DOM, a margin-top is added to create more white space.

content-area--wide

The --wide suffix increases the max-width of the content-area.

content-area--full-width

The --full-width suffix negates the default max-width of a content-area.

content-area--divider-bottom

content-area--divider-top

Adding a content-area divider helper class will add a border-top or border bottom to the content-area. To semantically create a content division, make sure the content-area is an appropriate HTML5 element (e.g. section) or use a <hr /> in place of the divider helper class.

content-area--offset

When content areas directly follow a previous content area in the DOM, a margin-top is added to create more white space.

content-area.content-area--callout

Adding a content-area--divider helper class will add a border-top to the content-area. To semantically create a content division, make sure the content-area is an appropriate HTML5 element (e.g. section) or use a <hr /> in place of the divider helper class.
